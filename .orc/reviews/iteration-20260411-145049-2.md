## Findings

### [P0] Race condition in ProcessRunner terminationHandler setup
- **File**: Orc/Core/Providers/Source/ProcessRunner.swift:110
- **Category**: bug
- **Issue**: `process.terminationHandler` is set after `process.run()` (line 88 runs the process, line 111 sets the handler). Foundation's process monitoring dispatch source checks `terminationHandler` when the process exits — if the process terminates before the handler is installed, the callback is never invoked and `withCheckedContinuation` never resumes, causing a permanent hang. The race window is the ~10 synchronous statements between `run()` and the handler assignment. While unlikely for long-running AI agent commands, immediate-exit processes (`/bin/true`, `exit 0`, or a failed exec) can hit this.
- **Fix**: Move `process.run()` into the `withCheckedThrowingContinuation` closure, after `terminationHandler` is set. This guarantees the handler is installed before the process starts, eliminating the race entirely. Use `withCheckedThrowingContinuation` so `run()` errors propagate correctly.
- **Status**: Fixed
- **Action**: Restructured to set terminationHandler before run() inside a single withCheckedThrowingContinuation block. Timeout task creation moved into the same closure after successful launch.
