### [P0] on_failure: continue in implement-task.yaml masks child workflow failures
- **File**: .orc/workflows/implement-task.yaml:14
- **Category**: bug
- **Issue**: The `implement` node has `on_failure: continue`, but it is the sole node in the child workflow. When the agent fails (crashes, timeout, etc.), the engine's `on_failure: continue` strategy prevents setting `runFailed`, so the child workflow exits with status `.completed`. The parent workflow's `software-engineer` node then reports `.completed`, causing the `fix-loop` guard (`when: "{{software-engineer.status}} == 'completed'"`) to pass, running the fix-loop on failed/absent changes. The parent already has `on_failure: continue` on the `software-engineer` node, which is the correct level to handle the failure.
- **Fix**: Remove `on_failure: continue` from the `implement` node in `implement-task.yaml`. The default `on_failure: stop` correctly propagates the failure to the parent, where the parent's own `on_failure: continue` handles it.
- **Status**: Fixed
- **Action**: Removed `on_failure: continue` from the `implement` node. Failure now propagates correctly: child fails → parent node fails → parent's `on_failure: continue` lets workflow proceed → `fix-loop` guard correctly skips.
